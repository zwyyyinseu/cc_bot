# Claude Vibecoding Bot 使用指南 🤖

> 在手机飞书上直接与服务器 Claude 对话，随时随地进行代码学习、开发和调试。

---

## 快速开始

1. 打开飞书，搜索 **Claude Vibecoding** 机器人
2. 直接发消息即可，Claude 会自动回复
3. 上下文自动保持记忆，连续对话无需重复说明

---

## 命令列表

| 命令 | 说明 |
|------|------|
| `/new 标题` | 创建新对话并切换（标题可选，默认"新对话"） |
| `/list` 或 `/ls` | 查看所有对话列表 |
| `/switch 序号` | 切换到指定对话（用 /list 中的序号） |
| `/del 序号` | 删除指定对话 |
| `/stop` | 终止当前正在执行的 Claude 任务 |

---

## 多对话管理

### 为什么需要多对话？

每个对话有独立的 **上下文记忆** 和 **工作目录**，适合不同项目/主题：
- 对话1：PCIe 学习 → Claude 记住你之前问的 PCIe 相关内容
- 对话2：DMA 开发 → Claude 记住 DMA 项目上下文
- 切换回对话1 → 自动恢复之前的讨论

### 对话列表示例

```
用户: /list

Bot:
📑 对话列表（共 3 个）

▸ 1. PCIe学习      [活跃] ✓ 2分钟前
  2. DMA开发          ✓ 1小时前
  3. 调试笔记         ○ 昨天

💡 /switch 序号 切换  /del 序号 删除  /new 标题 新建
```

> ✓ = 有历史上下文（可 --resume）  ○ = 新会话

### 常用操作

**创建新对话：**
```
/new PCIe学习
/new DMA开发
/new            ← 无标题时默认"新对话"
```

**切换对话：**
```
/switch 2       ← 切换到序号2的对话
```

**删除对话：**
```
/del 3           ← 删除序号3的对话
```

## 使用技巧

### 💡 让 Claude 帮你导航

不知道该看哪个文件？直接问：
```
我想学习 PCIe DMA 的实现，应该从 rtl 目录的哪个文件开始？给我一个学习路线
```

### 💡 连续追问（上下文记忆）

Claude 会记住之前的对话，你可以连续追问：
```
第1条：分析 pcie_us_axi_dma.v 的功能
第2条：它里面的 RD 通路具体怎么工作的？   ← Claude 知道你在说哪个文件
第3条：和 WR 通路有什么区别？              ← Claude 知道上下文
```

### 💡 多项目多对话

不同项目用不同对话，上下文不混：
```
/new PCIe学习          ← 开启新对话
分析 pcie_us_if.v      ← 在这个对话里讨论 PCIe

/new DMA开发           ← 开启另一个新对话
帮我写一个 DMA controller ← 在这个对话里做开发

/switch 1              ← 切回 PCIe 学习，继续上次的话题
```

### 💡 让 Claude 画时序图

```
用文字画出 pcie_us_if_cq.v 的 CQ 接口握手时序图
```

### 💡 紧急停止

如果 Claude 跑偏了或在执行危险操作：
```
/stop   ← 立即终止当前任务（不会切换对话）
```

---

## 注意事项

1. **回复延迟**：Claude 需要几秒到几十秒处理，复杂任务可能更久，期间会显示"正在思考..."
2. **文件路径**：使用服务器上的绝对路径，如 `/home/zzzwy/workspace/...`
3. **上下文长度**：对话过长时记忆可能被截断，用 `/new` 开启新会话重置
4. **工作目录**：Claude 的工作目录是 `/home/zzzwy/workspace/cc_bot/workspace/`，但它可以读取服务器上任何你有权限访问的文件
5. **消息格式**：Claude 的回复以飞书卡片形式展示，支持 Markdown 格式
6. **对话切换**：切换对话时会自动终止当前运行的 Claude 进程
